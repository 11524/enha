## 1. 개요 ¶

[IBM](IBM.md)에서 개발한 운영체제.  
현재 개발 및 지원하는 회사는 Serenity Systems International(<http://www.ecomstation.com>)

  

[[edit](http://rigvedawiki.net/r1/wiki.php/OS/2?action=edit&section=2)]

## 2. 명칭 ¶

정식명칭은 'IBM Operating System/2', 혹은 'Microsoft Operating System/2'. 뒤에 붙은 '/2'는
'차세대'를 뜻한다.

  

IBM PC DOS와 MS DOS의 관계처럼, IBM이 공급할 때는 'IBM OS/2'. MS가 OEM으로 공급할 때는 'MS OS/2'라는
명칭을 썻다. 버전 2 이상은 IBM판만 존재하며, 버전 3부터 'OS/2 Warp'라고 불리게 된다.

  

[[edit](http://rigvedawiki.net/r1/wiki.php/OS/2?action=edit&section=3)]

## 3. 특징 ¶

사실 기능면에서 보자면 가히 '당대 최강'이라고 부를 만 하다.

  

  * 1.3 이하는 80286 이상 CPU에서 쓰는 16비트 OS, 2.0 이상은 80386 이상에서 사용하는 32비트 OS. [PowerPC](PowerPC.md)용도 개발되었다.
  * 파일 시스템 : DOS와 같은 FAT를 썼으며, HPFS가 추가된다.
  * 선점형 멀티태스킹을 처음부터 지원했다.
  * [GUI](GUI.md)  

    * 버전 1은 프리젠테이션매니저 ( PM )`[1]`
    * 버전 2 이상은 객체지향의 워크플레이스쉘 ( WPS )
  * DOS 호환 환경  

    * 버전 2 이상은 복수의 다중 가상 DOS 머신(MVDM)으로 DOS 호환환경을 만들었다. 가상86 모드를 사용하여 여러 DOS 머신을 선점형 멀티 태스킹 환경에서 실행시킨다. BIOS와 일부 드라이버의 실제 처리 코드를 다른 주소 공간 MVDM에 그 호출 코드를 배치해서 DOS 응용 프로그램에서 사용할 수 있는 기본 메모리를 넓게 확보 할 수 있었다. 따라서 Windows의 DOS 명령 프롬프트, 나아가서는 DOS 자체와 비교해도 응용 프로그램의 동작을 안정시킬 수 있는 경우가 존재했다.
  * Windows 호환 환경(WIN-OS/2) : 에뮬레이션이 아니라 마이크로소프트의 라이센스를 포함해서, Windows 본체 모듈을 OS/2 MVDM에서 실행하고 여러 WIN-OS/2 어플리케이션을 가동 할 수 있었다. 호환성 및 여러 응용 프로그램을 실행시켜 때의 안정성이 매우 우수했다. 버전 2는 Windows 3.0, 버전 2.1 이상은 Windows 3.1에 해당하는 코드를 실행할 수 있었다.  

IBM은 자신들의 IBM PC가 [마이크로소프트](%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8.md)에 그 [운영체제](%EC%9A%B4%EC%98%81%EC%B2%B4%EC%A0%9C.md)를
의존하고 있는 현실에 위기의식을 느끼기 시작했고, 그러한 현실을 극복하기 위해 새로운 운영체제인 OS/2 개발을 마이크로소프트에 맡긴다.
**어?**`[2]``[3]` 그래서 버전 1.x 대까지는 [마이크로소프트](%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8.md)와 공동으로 개발하는 모습을 갖추고 있었다. 그러나, [빌게이츠](%EB%B9%8C%20%EA%B2%8C%EC%9D%B4%EC%B8%A0.md)는 OS/2가 마이크로소프트를 엿먹이기 위한
IBM의 계획의 일부라는 것을 알고 있었고, 겉으로는 OS/2에 협력하는 척하면서
[윈도우즈](%EC%9C%88%EB%8F%84%EC%9A%B0%EC%A6%88.md)의 개발에 박차를 가한다.`[4]` 결국 계약이
깨지면서 마이크로소프트는 [윈도우즈](%EC%9C%88%EB%8F%84%EC%9A%B0%EC%A6%88.md)에 전념하여 윈도우즈
3.x 및 윈도우즈 NT 시리즈를 만든다. 이 사건 때문에 IBM은 마이크로소프트와 사이가 매우 나빠졌었는데, IBM이 PC사업을 완전히
처분하고 중국의 [레노보](%EB%A0%88%EB%85%B8%EB%B3%B4.md)에 넘길때까지 결코 IBM 노트북이나 키보드에 윈도우
버튼을 넣지 않았던 이유가 이 때문이라는 소문도 있을 정도였다.

  

[1992년](1992%EB%85%84.md)에 나온 OS/2 2.0 부터는 네이티브 32비트 OS로 만들어져서 당시 주 경쟁자였던
윈도우즈 3.x에 비해 32비트 CPU를 완벽하게 지원하였고, 선점형 멀티 태스킹 등 높은 성능과 안정성을 자랑하였으며`[5]`,
강력하면서도 이용하기 편한 객체지향 사용자 인터페이스 등으로 크게 앞섰다는 평가를 받았는데,
[1996년](1996%EB%85%84.md)에 나온 OS/2 4.0은 기술적으로 수년 후에 나온 윈도우즈 **2000**보다도
앞섰다는 평가를 내린 사람도 있다. 하지만 이미 그때부터 이런저런 [삽질](%EC%82%BD%EC%A7%88.md) 및 높은 시스템
사양으로 인해 시장 점유율은 그리 높지 않았고, 당연히 회사들은 널리 퍼진 윈도우즈용 프로그램만 죽어라 만들고 팔았으므로(...) 응용
프로그램 부족이 고질적인 문제였다.

## 4. 버전 ¶

[[edit](http://rigvedawiki.net/r1/wiki.php/OS/2?action=edit&section=5)]

### 4.1. OS/2 1.x ¶

  * 1.0 : GUI는 탑재되어 있지 않고, OS/2 환경과 DOS 환경을 키보드로 스위칭 한다. OS/2 환경에서는 처음부터 선점형 멀티태스킹을 할 수 있었다.
  * 1.1 : GUI로 OS/2 프리젠테이션 매니저(PM)가 탑재. 외관은 Windows 2.x과 비슷하다.
  * 1.2 : [마이크로소프트](%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8.md)가 손을 떼고, IBM이 주도하는 개발이 되었다. 더욱 깔끔해진 외관은 Windows 3.x와 거의 비슷하며, Windows 3.x의 UI 디자인은 사실 OS/2 1.2가 먼저 도입했다.`[6]`
  * 1.3 : 1990년 발매. 더욱 경량화 되었다.  

1.3 발매 이후, IBM은 OS/2 2.0을 개발하게 되고 마이크로소프트는 OS/2 3.0을 주로 개발하게 되었다. 당시 개발하던 OS/2
3.0은 이후 [Windows NT](Windows%20NT.md)로 개명된다. Windows 3.0을 출시하면서 IBM과 긴장이
발생하여 공동 개발이 해소된다.

  

[[edit](http://rigvedawiki.net/r1/wiki.php/OS/2?action=edit&section=6)]

### 4.2. OS/2 2.x ¶

  * 2.00 : 1992년 3월 31일 발매, 세계 최초의 개인용 컴퓨터 용 32 비트 OS. Windows 3.0 호환 환경(WIN-OS/2)과 다중 DOS호환환경(MVDM)을 탑재. 통합 플랫폼으로서 하나의 완성형을 나타냈다. 객체지향 [GUI](GUI.md) 워크플레이스쉘(Workplace Shell, WPS)을 표준 환경으로 하였다.  

    * 워크플레이스쉘은 CORBA 준거의 개체 간 통신 기술/ SOM(System Object Model)/DSOM(Distributed SOM) 위에서 구축되어 있었다. SOM은 객체지향이 아닌 OS에서 언어에 거의 의존하지 않고 객체 지향 기능을 제공하는 오브젝트 관리를 위한 개발 환경이다. 이 기술 배경은 개체의 동적 추적 등이 가능하다.
  * 2.1 : WIN-OS / 2가 Windows 3.1에 해당하도록 변경. Windows 3.1용 응용 프로그램을 돌릴 수 있게 된다.  

1993년 9월, IBM과 마이크로 소프트의 소스 코드의 상호 공개 계약이 만료, 이후 OS / 2 및 Windows NT는 완전히
차별화되어 각각 다른 방향으로 발전하게 된다.

  

1994년 10월, IBM은 모토로라 와 함께 [PowerPC](PowerPC.md)를 사용한 플랫폼, PowerPC Reference
Platform (PReP)을 발표했다. PReP 위해 마이크로 커널 에서 여러 OS를 동작시키는 Workplace OS 의 개발을 IBM은
표명했지만 결국은 OS/2 for PowerPC 를 만들어내는 데 그쳤다. 또한 이 때부터 애플 컴퓨터, 노벨과 함께
OpenDoc(마이크로소프트의 OLE와 거의 같은 기능을 고기능 멀티 플랫폼 화 한 것)의 개발에 착수. Warp4에 탑재되었지만, 후년
[Java](Java.md)에 밀려 대중화에는 이르지 않았다.

  

[[edit](http://rigvedawiki.net/r1/wiki.php/OS/2?action=edit&section=7)]

### 4.3. OS/2 Warp 3.x ¶

1995년 등장한 OS/2 Warp 3.0은 윈도우즈 95와 같은 시기에 등장해서 윈도우즈의 유력한 라이벌로 주목받았다. 역시
IBM답게(?) 32bit 환경을 구축한 마이크로소프트에서 Win16 코드판권을 구입해서 탑재한 덕분에 OS/2에서 윈도우즈 3.xx용
프로그램을 바로 돌릴 수 있게 되었다 이로써 프로그램 부족 문제를 어느 정도 해소할 수 있지만, 당연히 **개발사들이 OS/2용 프로그램을
개발할 필요성이 줄어들었다.(…)** 결과적으로 IBM은 [자충수](%EC%9E%90%EC%B6%A9%EC%88%98.md)를 둔
셈이니 캐[안습](%EC%95%88%EC%8A%B5.md)(…).`[7]` 거기다, IBM 의 생각보다 32 bit 로 소프트웨어로의
전환이 워낙 빨리 이뤄진 바람에 [망했어요](%EB%A7%9D%ED%96%88%EC%96%B4%EC%9A%94.md) 덕택에 PC-
DOS 를 OS/2 상에서 돌린다든지 윈도 3.11용 소프트웨어를 안정적인 OS/2 위에서 가상화하여 굴리는 방식으로 돌리는 사람도
있었으나, 당시 컴퓨터 사양은 OS/2만 돌리는 것만으로도 상당히 빡셌는데, **램값=금값**이라고 불릴정도로 무지막지했던 당시의 메모리
가격이 시너지 효과를 발휘해서 역시 망했어요.`[8]` 그래도 불굴의 IBM은 4.0 멀린까지 출시했으나 이제는 아무도 관심을 갖지
않았고(...) 마케팅 문제 및 응용 프로그램 부족으로 사실상 사장되었다. 현재는 단종되어 eComStation으로 후속 버전이 나오고
있다.

  

[한국](%ED%95%9C%EA%B5%AD.md) 발매시 베타 버전을 잡지사에 번들로 제공하기도 했으며, 정품 버전은 번들 소프트웨어를
빵빵하게 끼워주기도 했다.`[9]` Warp 3의 소비자 가격은 약 1만원 초반대. 하지만 영어를 **일본어로 번역하고, 이걸 다시 한국어로
번역**하는 과정에서 번역의 질이 [발번역](%EB%B0%9C%EB%B2%88%EC%97%AD.md) 수준이 되고 말았다. 현재
한국에서 구할 수 있는 거의 유일한 OS/2 관련 자료는 [여기#](http://ecomstation.co.kr/doc.php)로.
한국에서의 개인 사용자는 극히 적은 편이나, 안정성이 중요한 산업용 장비에는 아직도 이런 운영체제가 돌아가고 있다. 2000 년대 중반까지만
해도 은행의 ATM 시설 상당수가 OS/2 로 돌아가고 있었다. 하지만 지금은 ATM도 터치패널 및 더 빠른 성능을 필요로 하게 되어서
임베디드 XP 및 유닉스 환경으로 업그레이드 하였다.

### 4.5. eCS ¶

2001년 7월에 WSeB와 Convenience Pak 1에 기반한 eCS 버전 1.0이 출시 되었다. 저널링 파일시스템과 몇 가지 사항이
개선되었다.

  

2004년에 eCS 1.2가 발매되고, 몇 년간 베타판만 신나게 제공하다가 2010년 5월 드디어 eCS 2.0 GA가 발매되었다.
부팅가능한 [JFS](JFS.md), [ACPI](ACPI.md), CIFS/SMB 지원과 더 많은 소프트웨어
지원([파이어폭스](%ED%8C%8C%EC%9D%B4%EC%96%B4%ED%8F%AD%EC%8A%A4.md)나
[오픈오피스](%EC%98%A4%ED%94%88%EC%98%A4%ED%94%BC%EC%8A%A4.md) 등의 상향 버전 지원) 등이
추가되었다. <del>하지만 다른 [운영체제](%EC%9A%B4%EC%98%81%EC%B2%B4%EC%A0%9C.md)는 다 이미
지원하는 기능이잖아? [우린 안될꺼야 아마](%EC%9A%B0%EB%A6%B0%20%EC%95%88%EB%90%A0%EA%BA%BC%EC%95%BC%20%EC%95%84%EB%A7%88.md)...</del>
