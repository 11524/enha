## 1. 개요 ¶

Windows Server 2003은 마이크로소프트에서 만든 [Windows XP](Windows%20XP.md)기반의 서버
운영체제다. [Windows 2000](Windows%202000.md)에도 서버 에디션은 있었지만, 독립되어 출시된것은 처음이다.

  

[[edit](http://rigvedawiki.net/r1/wiki.php/Windows%20Server%202003?action=edit
&section=2)]

## 2. 에디션 ¶

  * 웹 에디션
  * 스텐다드 에디션
  * 엔터프라이즈 에디션
  * 데이터센터 에디션  

각각의 에디션별 차이는 서버 동시 접속자수의 차이 이외에 큰 차이는 없다. <del>물론 가격차이는 있다</del>

  

[[edit](http://rigvedawiki.net/r1/wiki.php/Windows%20Server%202003?action=edit
&section=3)]

## 3. 새로운 기능 ¶

  * 인터넷 정보 서비스 (IIS) v6.0 포함
  * 방화벽 내장&대부분 서비스 사용안함.  
보안성능 향상을 위해서라고 알려져 있으며, 서비스 대부분이 비활성화 상태이기에 동일 사양에서 XP보다 빠르다.

  * 메시지 쿼링 개선
  * 역할 관리 도구
  * 계획에서 클래스를 제거하거나 여러 개의 디렉터리 서버의 프로그램을 실행하는 기능과 같은 액티브 디렉터리의 개선
  * 그룹 정책 관리 개선
  * 파일 섀도잉 백업의 도움으로 열려 있는 파일의 백업을 허용하는 기능 등을 포함하여 디스크 관리 개선
  * 마이크로소프트가 차기 버전의 윈도의 명령 줄로 생각해 두었던 스크립팅 및 명령 줄 도구 개선
  * 하드웨어 기반의 감시 타이머 포함  

[[edit](http://rigvedawiki.net/r1/wiki.php/Windows%20Server%202003?action=edit
&section=4)]

## 4. Windows Server 2003 R2 ¶

Windows Server 2003을 기반으로, 약간의 개선을 거친 제품이다.

  

  * 브랜치 오피스 서버 관리
  * 파일, 프린터에 중점적인 관리 도구
  * 강화된 분산 파일 시스템 (DFS) 이름 공간 관리 인터페이스
  * RDC를 사용하여 WAN 데이터를 더 효율적으로 복제할 수 있음
  * 아이덴티티 및 접근 관리
  * 사용자 접근 기록
  * 엑스트라넷 응용 프로그램 접근의 중점적인 관리
  * 기억 장치 관리
  * 파일 서버 리소스 관리자
  * 할당 관리 강화
  * 파일 스크리닝을 통한 제한되는 파일 형식 제한
  * SAN을 위한 기억 용량 관리
  * 서버 가상화
  * 새로운 라이선스 정책: 엔터프라이즈 에디션의 경우 4 개의 가상 인스턴스 허용, 데이터센터 에디션의 경우 제한 없음
  * 유닉스 기반 응용 프로그램 추가 기능을 위한 유틸리티와 SDK
  * 기본 유틸리티
  * SVR-5 유틸리티
  * 기본 SDK
  * GNU SDK
  * GNU 유틸리티
  * 펄 5
  * 비주얼 스튜디오 디버거 애드인

## 5. 지원 기간과 서비스 팩 ¶

메인스트림 지원은 2010년 7월 13일에 [Windows 2000](Windows%202000.md)과 함께 종료되었다. 하지만
연장지원은 XP보다 조금 긴 2015년 7월 14일 까지다. 서비스팩은 2까지 출시되었다. 어쩔 수 없이 저사양 데스크탑 PC에 XP를 써야
한다면 이쪽을 알아보는 편도 나쁘진 않다.

  

[[edit](http://rigvedawiki.net/r1/wiki.php/Windows%20Server%202003?action=edit
&section=6)]
