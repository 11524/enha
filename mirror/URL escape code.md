이런 거.

URL Encoding에 사용된다. 그 정체는 #뒤에 [아스키코드](%EC%95%84%EC%8A%A4%ED%82%A4%20%EC%BD%94%EB%93%9C.md)의 hex(16진수)값을 붙여준
것.

URL은 스페이스(space)를 포함할 수 없기 때문에, 스페이스는 %20으로 대체되는 것이 보통이다. 지금 웹 브라우저의 주소창을 확인하면
"…/URL escape code"가 아닌, "…/URL**%20**escape**%20**code"라고 되어 있을 것이다. URL
escape code가 사용되는 가장 흔한 예.

특정 동작을 수행할 목적으로(?) 특수문자를 사용하는 경우 코드 충돌을 피하기 위해 URL escape code를 사용해야 한다. 리그베다
위키의 경우, 문서의 URL 뒤에 `?action=edit` 을 붙이면 항목을 편집할 수 있으니 항목명에 있는(따라서 URL에 포함되는)
?는 이 기능과 충돌할 수 있다.`[1]` 이 때문에 위키 내에서 링크를 만들 때 항목명에 ?가 있다면 그것을 %3F로 바꿔줘야 하는
것이다. (예 : [도대체 왜?인구단](%EB%8F%84%EB%8C%80%EC%B2%B4%20%EC%99%9C%3F%EC%9D%B8%EA%B5%AC%EB%8B%A8.md))`[2]`

&는 %26으로(예 : [록맨 &포르테](%EB%A1%9D%EB%A7%A8%20%26%20%ED%8F%AC%EB%A5%B4%ED%85%8C.md))`[3]`, #은
%23으로(예: [C#](C%23.md))`[4]`... 하여간 많다.

항목명이 영단어인데 그것이 명령어 등과 겹칠 경우에는 알파벳도 URL escape code로 풀어주어야 하는 수가 있다. 항목명 GOTO로
가기 위해 "[%47%4F%54%4F](GOTO.md)"처럼 링크를 걸어주면 충돌을 피할 수 있다. 하지만 리그베다 위키에는
`["GOTO"]`라는 훌륭한 회피수단(...)이 있으므로 굳이 URL escape code로 풀어주지 않아도 된다. 혹시 이런 기능을
지원하지 않는 곳에서 문서를 작성하다가 오류가 난다면 그 때 써먹을 수는 있을 것이다.

리그베다 위키에서는 링크를 걸 때 항목명의 특수문자를 URL escape code로 풀어주거나, 다른 식으로 신경을 써 주어야 하는 경우
항목에 다음 틀이 붙는다.

**주의. 기술적인 문제로 인해 본 문서로 오는 링크를 만들 때 주의를 요합니다.**  
  
기술적인 문제로 인해 다른 문서에서 이 문서로 오는 링크를 만들 때 [[문서명]]을 사용할 수 없거나, 특별한 처리를 필요로 합니다.
**[[문서명|]]**을 이용해 주세요. 단, 제목에 특수문자가 들어가는 문서의 경우 그 자리에 아스키 코드값을 넣어야 할 수 있습니다(예:
[[C%23|C#]]).
