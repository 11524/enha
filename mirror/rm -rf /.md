## 2. 설명 ¶

유닉스의 rm명령어는 파일을 삭제하는 명령어이다. 뒤에 -rf는 스위치로 -r(`\--recursive`, 재귀) 하위 디렉터리 및 파일을
하나하나 깔끔하게 다 지워버리고 마지막엔 자기 디렉터리로 돌아와 자기 자신까지 삭제하며 -f`[2]``(--force)`는 <del>[묻지도따지지도 않고](%EC%9D%B4%EC%88%9C%EC%9E%AC.md)</del>확인 절차를 생략하며, /는 리눅스 파일 시스템의
최상위 디렉터리이다. 즉 윈도에서의 **Format C:**수준 이상(……). 물론 NT계열 윈도우에서는 명령이 제대로 동작한 경우 시스템
드라이브라는 <del>불평</del>오류를 뱉어내며 안 먹히지만`[3]`, 유닉스의 경우는 **마운트 하고있던 모든 장치의 내용마저
날려먹는다**. <del>-rm -rf 실행할때 야동 외장하드가 꽃혀있었으면? 꽃망</del> 그리고 여담이지만,안드로이드에서 터미널
에뮬레이터로 해봐라. rm -rf를 치는순간 ~~~프로그램이 종료되었다는 말과 함께 자신의 멘탈까지 종료되는 현상을 볼 수도 있다.

  

단 윈도우의 format c:와 유닉스/리눅스의 rm -rf는 개념은 다르다. rm은 삭제 명령어가 맞지만, format c:는 c
드라이브를 포맷하라는 명령어이지 삭제 명령이 아니다. 포맷 과정에서 삭제가 이루어지기에 결과가 비슷해지는 것이다. 윈도우 커맨드 라인에서는
삭제 명령어만으로는 rm -rf 같은 명령을 쓸 수가 없는데, 윈도우의 삭제 명령어인 del은 폴더를 못 지우기 때문이다. 폴더를 지우기
위해서는 rmdir 명령을 써야 하는데, 폴더 안에 파일이 있으면 못 지운다. 즉 윈도우 커맨드 라인에서는 rm -rf와 완전히 개념이 같은
명령어는 존재하지 않는다.

## 3. 사용할 경우 생겨 나는 일 ¶

따라서 그냥 모르고 시전하면 [YOU JUST ACTIVATED MY TRAPCARD](YOU%20JUST%20ACTIVATED%20MY%20TRAP%20CARD.md) \+
[망했어요](%EB%A7%9D%ED%96%88%EC%96%B4%EC%9A%94.md)(....)를 띄워주며, 보통 터미널상에서
디렉토리를 삭제할때 rm -rf를 사용 `[4]`하는데, 까딱 잘못 치면 묻지도 따지지도 않고 안의 내용을 지워<del>주면서
머릿속도</del> 포맷해준다(……) 잘못 사용하는 대표적인 예로는

  

> cd /home/user/test  
rm -rf ./*

  
이런식으로 치려다가 삐끗해서 rm -rf /*을 쳐버리는 경우 심히 난감하다.  
**정말 다 날아간다. 머리속까지 싸그리 [포맷](%ED%8F%AC%EB%A7%B7.md)**. 설명이 더 [必要韓紙](%E5%BF%85%E8%A6%81%E9%9F%93%E7%B4%99.md)?

## 4. 트리비아 ¶

주의할 것. 게다가 앞에 sudo가 붙고 스위치로 **`"--no-preserve-root"`**가 붙으면
[끔살](%EB%81%94%EC%82%B4.md) 확정이다. 정 저걸 다 적기 싫다면 rm -rf /* 라고 입력해주면 된다.
